## Bảo mật

### CORS giới hạn origin (`server.ts`)

Trước: `app.use(cors())` — cho phép mọi origin.

Sau:
```ts
const allowedOrigins = process.env.ALLOWED_ORIGINS
  ? process.env.ALLOWED_ORIGINS.split(',').map(o => o.trim())
  : ['http://localhost:3000', 'http://localhost:24679'];
app.use(cors({ origin: ..., credentials: true }));
```

Deploy production cần set env `ALLOWED_ORIGINS=https://your-domain.com`.

### Helmet security headers (`server.ts`)

Thêm `helmet` middleware — tự động gắn các header:
- `X-Frame-Options: SAMEORIGIN`
- `X-Content-Type-Options: nosniff`
- `Strict-Transport-Security`
- `X-DNS-Prefetch-Control`
- v.v.

`contentSecurityPolicy: false` vì CSP đang quản lý riêng phía frontend (Vite).

### Giảm payload limit (`server.ts`)

`express.json({ limit: '50mb' })` → `'10mb'` (base64 ảnh thực tế ~5–8 MB).

### Ẩn thông tin nhạy cảm trong error log (`firebase.ts`)

`handleFirestoreError` chỉ log `email`, `emailVerified`, `providerInfo` khi `import.meta.env.DEV`.  
Production chỉ log `userId` — không lộ email/provider ra console.

## Hiệu năng React

### React.memo + useCallback cho PromptManager (`components/prompts/PromptManager.tsx`)

- Component wrap `React.memo` — bỏ qua re-render khi props không đổi.
- `addPrompt`, `handleRandomPrompt`, `removePrompt`, `updatePrompt` wrap `useCallback`.
- Các handler dùng functional update (`setPrompts(prev => ...)`) — không capture `prompts` stale.

### React.memo cho ImageCard (`components/results/ResultsDisplay.tsx`)

`ImageCard` (nội bộ) wrap `React.memo` — ngăn re-render toàn lưới khi state parent thay đổi nhưng `generatedImages` không đổi.

### Firestore onSnapshot thay polling (`hooks/usePendingUsersNotifier.ts`)

Trước: `setInterval` + `getCountFromServer` mỗi 45 giây.

Sau: `onSnapshot` Firestore realtime listener — cập nhật tức thì khi có user pending mới, không tốn request định kỳ. Cleanup `unsubscribe()` khi `isAdmin` thay đổi hoặc unmount.

## Code quality

### Utility `isAcceptedImageFile` (`utils/fileValidation.ts`)

```ts
export const ACCEPTED_IMAGE_TYPES = 'image/png, image/jpeg, image/webp';
export function isAcceptedImageFile(file: File): boolean {
  return file.type.startsWith('image/');
}
```

Dùng chung ở `ImageUploader`, `ReferenceImageUploader`, `MaskUploader` — thay `file.type.startsWith('image/')` lặp lại và hardcode string `accept`.

## Bug fix analytics (cùng ngày)

### activeUsers 30d trả về toàn 0 (`services/analyticsAggregation.ts`)

`aggregateTrendPoints` chế độ `30d` chỉ xử lý `generations` và `cost`, bỏ sót `activeUsers` — biểu đồ "Người dùng hoạt động" range 30 ngày hiển thị toàn 0.

Fix: thêm `Map<string, Set<string>>` per-day tương tự 8-week view, điền `dayMap` bằng `set.size` sau vòng lặp.

### Admin pagination gọi thừa Firestore (`server/repositories/analyticsAdminRepository.ts`)

Mỗi trang pagination fetch lại document cuối qua `db.collection(...).doc(lastId).get()` — 1 read thừa mỗi page 500 docs.

Fix: lưu `lastDoc: QueryDocumentSnapshot` từ snapshot, dùng `startAfter(lastDoc)` trực tiếp.
