## Analytics aggregation

### Kiến trúc 3 tầng

```
repositories/analyticsRepository.ts   ← Firestore I/O (pagination)
services/analyticsTypes.ts            ← Types dùng chung
services/analyticsAggregation.ts      ← Pure functions (không I/O)
services/analyticsService.ts          ← trackEvent, loadMonthlyDashboardBundle, …
```

### Hàm aggregation chính (`services/analyticsAggregation.ts`)

- `monthBounds`, `monthBoundsFromKey`
- `buildMonthlyAnalyticsFromEvents`, `buildMonthlyErrorBreakdownFromEvents`, `buildMonthlyTokenStatsFromEvents`, `buildMonthlyTopModelStatsFromEvents`
- `buildMonthlyDashboardBundleFromEvents`
- `buildMonthlyRollupDocument` — payload doc `analytics_monthly_rollups/{YYYY-MM}` (`version: 1`)
- `aggregateTrendPoints` — dùng bởi `hooks/useTrendData.ts`
- `aggregateHistoryCountsByUid` — dùng bởi `UserHistoryCountsPanel.tsx`
- `isDashboardRollupPayload`, `normalizeErrorBreakdownFromRollup`

### Luồng đọc dashboard tháng

1. Đọc `analytics_monthly_rollups/{monthKey}` nếu `version === 1`.
2. Nếu thiếu/sai version: quét `analytics_events` theo trang (`ANALYTICS_EVENTS_PAGE_SIZE = 500`).
3. Gộp KPI + error breakdown + token + top model qua aggregation.

### Job rebuild rollup (server, Admin SDK)

- `server/repositories/analyticsAdminRepository.ts` — quét events + ghi rollup
- `server/lib/monthlyRollupBuilder.ts` — `rebuildMonthlyAnalyticsRollup(monthKey)`

Chưa có HTTP route công khai; gọi từ script/Cloud Scheduler khi cần.

### Tests

- `services/analyticsAggregation.test.ts`
- `server/lib/rateLimit/rateLimit.test.ts`

Chạy: `npm test`

## Rate limit phân tán (Cloud Run)

### Vấn đề

In-memory `Map` trên từng instance Cloud Run → user có thể vượt 10 req/phút khi load balancer chuyển instance.

### Giải pháp

`server/lib/rateLimit/`:

| File | Vai trò |
| ---- | ------- |
| `config.ts` | `RATE_LIMIT_BACKEND`, 10 req/phút, window 60s |
| `memoryStore.ts` | Dev / fallback |
| `firestoreStore.ts` | Transaction trên `rate_limit_windows/{userId_windowIndex}` |
| `index.ts` | Production → Firestore; lỗi → fallback memory |

- `POST /api/rate-limit` — handler **async** (`postRateLimit`).
- Env: `RATE_LIMIT_BACKEND=memory|firestore` (mặc định production: `firestore` khi `NODE_ENV=production`).
- `server/lib/rateLimitStore.ts` — re-export deprecated, trỏ `server/lib/rateLimit`.

## Cấu trúc backend `server/`

```
server/
  firebaseAdmin.ts          # Admin SDK, named Firestore DB
  types.ts
  middleware/authenticate.ts
  routes/
    index.ts                # Đăng ký /api/*
    rateLimit.ts
    generate.ts
    providerTest.ts
  lib/
    resolveProvider.ts
    rateLimit/              # (xem trên)
    monthlyRollupBuilder.ts
  repositories/
    analyticsAdminRepository.ts
```

Entry: `server.ts` (root) — Express, static `dist/`, `registerApiRoutes`.

**Dockerfile (production stage):**

```dockerfile
COPY server.ts firebase-applet-config.json ./
COPY server ./server
```

Thiếu `COPY server ./server` → container crash, không listen `PORT=8080`.

## Firebase client persistence

### Trước

`getFirestore()` + `enableMultiTabIndexedDbPersistence(db)` — deprecated warning Firebase 12.10.

### Sau (`firebase.ts`)

```typescript
initializeFirestore(app, {
  localCache: persistentLocalCache({
    tabManager: persistentMultipleTabManager(),
  }),
}, firestoreDatabaseId);
```

Fallback `getFirestore()` nếu init cache lỗi (private mode, xung đột IndexedDB).

| Chunk | ~Kích thước (min) |
| ----- | ----------------- |
| `firebase-firestore-vendor` | ~350 kB |
| `firebase-auth-vendor` | ~112 kB |
| `firebase-misc-vendor` | ~80 kB |

## GA4 và cảnh báo trình duyệt

### `ERR_BLOCKED_BY_CLIENT` (googletagmanager.com)

- Ad blocker / privacy extension chặn `gtag/js`.
- **Không** ảnh hưởng app, Firestore analytics, generate.
- GA4 event trong `utils/gtagEvent.ts` no-op khi `window.gtag` không có (prod vẫn an toàn).

### Firestore deprecation (đã xử lý)

- Cảnh báo `enableMultiTabIndexedDbPersistence` → đã migrate sang `localCache` API.

## Deploy & troubleshooting

### Checklist deploy (thứ tự)

1. `npm run lint && npm run build && npm test`
2. `firebase deploy --only firestore:rules --project zvas-ai-image` (nếu đổi rules)
3. `gcloud run deploy ...` (Mode A nếu chỉ đổi code)

### Lỗi: Revision not ready — PORT 8080

**Triệu chứng:** Container failed to start and listen on PORT=8080.

**Nguyên nhân thường gặp:** Dockerfile không copy `server/` → import `./server/routes` fail.

**Xử lý:** Đảm bảo Dockerfile có `COPY server ./server`; rebuild deploy.

### Smoke test bổ sung

- Generate >10 lần/phút → 429 rate limit (Firestore backend trên prod).
- Admin → Analytics → **Yêu cầu dữ liệu** → trends + KPI.
- Console: không còn warning Firestore persistence (bản build mới).
